# Experiment 0000 — buildable research foundation

Status: **infrastructure pass; performance hypothesis untested**.

## Hypothesis

A sequential route simulator with explicit prediction-read accounting can reject
harmful residency policies before expensive model-runtime implementation.

## Contract

- Language: C++20 runtime and C kernel ABI.
- Precision floor: int4, enforced in model validation and the kernel ABI.
- Correctness: sanitizer build, unit tests, trace parser, and CLI smoke.
- Memory: synthetic fixtures only.
- Performance decision: none may be drawn from the synthetic trace.

## Commands

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Debug -DSTRATA_ENABLE_SANITIZERS=ON
cmake --build build --parallel
ctest --test-dir build --output-on-failure

./build/strata-sim --trace tests/data/route_trace_v1.txt --compare \
  --ram 8M --vram 4M --expert-bytes 4M --prefetch 4 --lease 8 --confidence 0.5
```

## Result

- 14/14 unit tests passed under AddressSanitizer and UndefinedBehaviorSanitizer.
- Both CTest targets passed.
- The intentionally constrained synthetic trace showed that prefetch can increase
  total NVMe bytes and displace useful pages. This validates the accounting path;
  it does not rank policies for a real model.

The simulator records demand misses, total and speculative NVMe bytes, useful and
wasted prefetch, H2D bytes, peer activation bytes, evictions, and strict cold-read
budget violations.

## Decision

Keep the foundation. Do not select a cache or predictor from synthetic data.

Next experiment: capture a real sequential route trace and compare no-prefetch
LRU/LFU against budgeted transition prediction and leases at identical RAM/VRAM
capacity. Require at least a 2x reduction in NVMe bytes before the novel policy is
promoted into the inference runtime.
